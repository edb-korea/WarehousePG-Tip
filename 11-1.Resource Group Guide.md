# WarehousePG Resource Group 가이드

## 1. Resource Group 설정 확인

### 1.1 Resource Group 기능 자체가 활성화되어 있는지 확인

```sql
SHOW gp_resource_manager;
```
결과가 `group`이면 리소스 그룹 기반 관리가 활성화된 상태입니다. (`queue`면 비활성화 = 예전 Resource Queue 방식)

### 1.2 생성된 리소스 그룹과 제한값(limit) 확인

모든 리소스 그룹의 제한 설정을 확인하려면 `gp_toolkit.gp_resgroup_config` 시스템 뷰를 사용합니다.

```sql
SELECT * FROM gp_toolkit.gp_resgroup_config;
```
컨커런시(동시 트랜잭션 수), 메모리 쿼터, CPU limit, IO limit 등이 그룹별로 나옵니다.

### 1.3 실시간 사용량(현재 얼마나 쓰고 있는지) 확인

```sql
-- 전체 상태
SELECT * FROM gp_toolkit.gp_resgroup_status;

-- 호스트별 CPU/메모리 사용량
SELECT * FROM gp_toolkit.gp_resgroup_status_per_host;

-- 호스트별 디스크 I/O 사용량
SELECT * FROM gp_toolkit.gp_resgroup_iostats_per_host;
```

### 1.4 어떤 role(사용자)에 어떤 그룹이 할당돼 있는지 확인

```sql
SELECT rolname, rsgname
FROM pg_roles r
JOIN pg_resgroup rg ON r.rolresgroup = rg.oid;
```

### 1.5 관련 GUC(서버 설정) 값들도 함께 확인

```sql
SHOW gp_resgroup_memory_policy;
SHOW gp_resgroup_memory_query_fixed_mem;
SHOW gp_resource_group_bypass;
```

> **참고:** `gp_resgroup_config` 뷰는 리소스 그룹 기반 자원 관리가 활성화된 경우에만 유효합니다. 위 쿼리들이 비어있거나 에러가 난다면 `gp_resource_manager`가 아직 `queue`(기본값)로 되어 있을 가능성이 높습니다.

## 2. Resource Group 최대 리소스로 변경하기

### 2.1 ALTER RESOURCE GROUP 기본 문법

```sql
ALTER RESOURCE GROUP rgroup_name SET CONCURRENCY 50;
ALTER RESOURCE GROUP rgroup_name SET CPU_MAX_PERCENT 100;
ALTER RESOURCE GROUP rgroup_name SET MEMORY_QUOTA 80;
ALTER RESOURCE GROUP rgroup_name SET IO_LIMIT 'pg_default: wbps=max, rbps=max, wiops=max, riops=max';
```

### 2.2 각 항목을 "최대"로 설정하는 방법

| 항목 | 의미 | 최대치 설정 |
|---|---|---|
| **CONCURRENCY** | 동시 실행 가능한 트랜잭션 수 | 범위는 `[0 .. max_connections]`이며, 0으로 설정하면 사실상 무제한(제한 없음). `SET CONCURRENCY 0;` |
| **CPU_MAX_PERCENT** | 사용 가능한 CPU 자원의 비율 | 최대값은 **100** |
| **MEMORY_QUOTA** | 쿼리가 소비할 수 있는 최대 메모리 비율(%) | 다른 그룹들과 합쳐서 100을 넘을 수 없으니, 남은 여유분만큼 최대치 설정 |
| **IO_LIMIT** | 디스크 read/write 처리량, IOPS 제한 | `wiops=max, riops=max` 등으로 지정하면 무제한 |

### 2.3 예시: `myuser_group`을 최대치로 변경

```sql
ALTER RESOURCE GROUP myuser_group SET CONCURRENCY 0;
ALTER RESOURCE GROUP myuser_group SET CPU_MAX_PERCENT 100;
ALTER RESOURCE GROUP myuser_group SET MEMORY_QUOTA 90;
ALTER RESOURCE GROUP myuser_group SET IO_LIMIT 'pg_default: wbps=max, rbps=max, wiops=max, riops=max';
```

### 2.4 확인

```sql
SELECT * FROM gp_toolkit.gp_resgroup_config WHERE groupname = 'myuser_group';
```

### 2.5 주의할 점

1. **CONCURRENCY**: `admin_group`은 CONCURRENCY를 0으로 설정할 수 없습니다.
2. **MEMORY_QUOTA 합계 제한**: 모든 리소스 그룹의 MEMORY_QUOTA 합이 100%를 넘을 수 없으므로, 다른 그룹(`default_group`, `admin_group`, `system_group` 등)이 이미 얼마를 쓰고 있는지 먼저 확인 후 남은 만큼만 배정하세요.
   ```sql
   SELECT groupname, memory_quota FROM gp_toolkit.gp_resgroup_config;
   ```
3. **CPU_MAX_PERCENT 합계**: CPU도 그룹별 합이 100%를 넘으면 경고가 뜰 수 있습니다 (버전에 따라 오버커밋 허용 여부 다름).
4. **즉시 반영되지 않을 수 있음**: ALTER RESOURCE GROUP으로 초기화한 메모리 할당 변경은 현재 실행 중인 쿼리의 자원 사용 때문에 즉시 반영되지 않을 수 있습니다. 운영 중 그룹이면 유휴 시간대에 변경하는 걸 권장합니다.
5. **"진짜 무제한"이 목적이라면** 차라리 해당 role을 `default_group`이 아닌 admin_group에 준하는 별도 그룹으로 두고 자원을 넉넉히 주는 게, 아예 리소스 그룹 관리를 끄는 것보다 안전합니다.
